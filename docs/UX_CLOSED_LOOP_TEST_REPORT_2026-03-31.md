### A. 环境与启动链路

#### A-01 首次初始化脚本不可用（高）
- 证据：`init_db.py` 第 103 行附近存在错误缩进，实测执行报 `IndentationError`。
- 用户影响：新环境无法按 README 完成初始化，直接阻断上线前验收与演示搭建。
- 解决方案：
  - 流程：修复缩进并把 `secrets` 导入补齐，增加“已存在管理员则退出”后的说明输出。
  - 交互：初始化脚本输出统一为“成功/跳过/失败”三态摘要。
  - 文案：错误提示明确到“行号 + 修复建议 + 示例命令”。
  - 容错：增加 `--dry-run` 检查模式，先验证配置再写库。

#### A-02 管理员重置密码脚本失效（高）
- 证据：`reset_admin_password.py` 第 11 行 `from main import pwd_context`，当前 `main.py` 无该符号。
- 用户影响：测试与演示时账号恢复链路断裂，发生“管理员无法登录”时无快速恢复手段。
- 解决方案：
  - 流程：改为 `from app.core.security import pwd_context`。
  - 交互：重置成功后输出用户名、更新时间、强制下次登录建议。
  - 文案：失败时提示“检查 DB 连通性/用户是否存在”。
  - 容错：增加参数校验（长度、复杂度）并返回非零退出码。

### B. 认证与权限体验

#### B-01 权限与认证错误文案中英混杂（中）
- 证据：
  - `app/api/dependencies.py` 第 27、34、55、61 行。
  - `app/api/routes/auth.py` 第 29 行。
- 用户影响：前端 toast 风格不一致，学生与教师对错误含义理解成本高。
- 解决方案：
  - 流程：建立统一错误码字典（如 `AUTH_INVALID`, `AUTH_EXPIRED`, `FORBIDDEN_ROLE`）。
  - 交互：前端按错误码映射统一中文可行动提示。
  - 文案：统一使用“原因 + 下一步操作”（如“登录已过期，请重新登录”）。
  - 容错：保留后端英文日志，但对前端输出标准化中文。

#### B-02 登录失败提示过于粗暴，误导定位（中）
- 证据：`frontend/src/views/Login.vue` 第 80 行固定提示“用户名或密码错误”。
- 用户影响：网络异常、服务不可用、401/500 都被误报为密码错误，用户无法自助处理。
- 解决方案：
  - 流程：按 HTTP 状态分支提示（401 凭证错误、5xx 服务异常、超时网络异常）。
  - 交互：登录按钮失败后提供“重试”与“检查服务状态”。
  - 文案：新增“无法连接服务，请确认后端已启动”。
  - 容错：指数退避重试一次并记录失败原因。

#### B-03 路由越权回退到根路径，易产生迷惑跳转（中）
- 证据：`frontend/src/router/index.ts` 第 120、123 行 `next('/')`。
- 用户影响：用户可能被重定向到与权限不匹配页面，形成“我点了 A 被送去 B”的困惑。
- 解决方案：
  - 流程：越权统一跳转到 `403` 解释页或 `profile` 并带 `from` 参数。
  - 交互：展示“你当前角色无权访问该页面 + 可访问入口”。
  - 文案：替换“需要管理员权限”为“当前账号为学生，仅可访问监控/实验/档案”。
  - 容错：保留原目标路由，角色切换后可一键返回。

### C. 监控与设备控制

#### C-01 监控数据加载失败时无用户级反馈（中）
- 证据：`frontend/src/components/Dashboard.vue` 第 413 行仅注释“交给拦截器”，第 522 行仅 `console.error`。
- 用户影响：页面可能空白或数据不更新，但用户不知是无数据还是网络故障。
- 解决方案：
  - 流程：引入页面级状态机（loading/empty/error/offline）。
  - 交互：在图表区域展示 `StatusPanel`，支持“重试”。
  - 文案：错误语义化（设备离线、无历史数据、权限不足）。
  - 容错：轮询失败后自动降级到慢轮询并提示重连中。

#### C-02 控制开关成功后全量刷新设备列表，导致闪烁与上下文抖动（中）
- 证据：`frontend/src/components/Dashboard.vue` 第 559 行、648 行在控制后调用 `fetchDevices()`。
- 用户影响：切换设备时可能被重置，用户感知为“开关后界面跳动”。
- 解决方案：
  - 流程：控制成功后仅 patch 当前设备状态，不做全量拉取。
  - 交互：开关进入短暂 disabled + loading，完成后恢复。
  - 文案：将“控制成功”细化为“水泵已开启/风扇已关闭”。
  - 容错：失败回滚保持，但加“点击重试”按钮。

#### C-03 导出日期使用 `toISOString` 存在时区偏移风险（中）
- 证据：`frontend/src/components/Dashboard.vue` 第 605、606 行。
- 用户影响：跨时区或本地时区边界时，导出日期可能前后错一天。
- 解决方案：
  - 流程：统一按本地日期格式化（`YYYY-MM-DD`）而非 UTC 字符串切片。
  - 交互：导出前展示“本地时区：UTC+8”提示。
  - 文案：出现空数据时提示“请检查日期与时区”。
  - 容错：后端接受时区字段并统一归一化。

#### C-04 学生导出设备数据权限与默认设备选择存在认知冲突（高）
- 证据：
  - `docs/ux_api_probe_results_2026-03-31.json` 第 758、765 行。
  - 后端班级设备校验：`app/api/routes/telemetry.py` 第 390、393 行。
- 用户影响：学生在监控页看到设备数据，但导出时报“非本班设备”，感知为系统错误。
- 解决方案：
  - 流程：学生端设备下拉仅显示可导出设备，或导出前先做权限预检。
  - 交互：导出按钮 hover 展示当前设备是否可导出。
  - 文案：拒绝提示追加“请切换到本班绑定设备后导出”。
  - 容错：若无可导出设备，给“联系教师绑定设备”的引导 CTA。

### D. 数据大屏展示

#### D-01 大屏错误仅写控制台，用户无感知（中）
- 证据：`frontend/src/views/DashboardDisplay.vue` 第 145、240、316 行仅 `console.error`。
- 用户影响：公开展示时若后端抖动，大屏看似“正常但不更新”，影响课堂可信度。
- 解决方案：
  - 流程：增加大屏状态条（在线/重连/离线）。
  - 交互：失败时保留最后一帧并叠加“数据更新时间 + 重试中”。
  - 文案：以展示场景友好方式提示“数据暂不可用”。
  - 容错：重试退避 + 断线后自动恢复。

#### D-02 图表趋势由随机扰动生成，存在“伪历史”误导（高）
- 证据：`frontend/src/views/DashboardDisplay.vue` 历史趋势由当前点随机生成（初始化图表段）。
- 用户影响：教师/评委可能把随机曲线误认为真实传感器历史，造成教学误导。
- 解决方案：
  - 流程：改为读取真实历史接口（或明确标注“示意趋势”）。
  - 交互：图表角标显示数据源（实时/历史/模拟）。
  - 文案：无历史时展示“暂无历史数据，不绘制趋势线”。
  - 容错：历史拉取失败时回退单点卡片，不绘制伪趋势。

#### D-03 设备 ID 固定化设计限制扩展（中）
- 证据：`frontend/src/views/DashboardDisplay.vue` 第 191 行 `const deviceId = 1`。
- 用户影响：多设备场景无法切换，演示内容与真实班级设备不一致。
- 解决方案：
  - 流程：支持 query 参数或配置项指定展示设备。
  - 交互：新增非侵入式设备切换（仅管理员可见）。
  - 文案：标题显示当前设备名称。
  - 容错：设备不存在时自动降级到默认设备并提示。

### E. 实验报告闭环

#### E-01 普通提交接口未限制“仅学生提交”（高）
- 证据：
  - 提交接口定义：`app/api/routes/assignments.py` 第 387 行。
  - 仅在文件提交接口做了学生限制：第 467 行。
  - 提交权限判断存在 `current_user.role != "teacher"` 例外：第 406 行。
- 用户影响：教师/管理员可通过普通提交接口写入学生提交流程，破坏业务边界。
- 解决方案：
  - 流程：普通提交接口统一加 `if current_user.role != 'student': 403`。
  - 交互：前端隐藏非学生提交入口并禁用 API 调用。
  - 文案：明确“仅学生可提交报告”。
  - 容错：记录异常调用审计日志。

#### E-02 学生统计为前端伪造值，误导进度判断（高）
- 证据：`frontend/src/views/Assignments.vue` 第 529 行按 60%/50% 人工估算。
- 用户影响：学生看到的“已提交/已批改”与真实数据不一致，损害信任。
- 解决方案：
  - 流程：改为后端返回真实统计字段。
  - 交互：统计卡显示“实时统计更新时间”。
  - 文案：未获取到统计时明确“统计计算中”。
  - 容错：统计接口失败时隐藏数值，展示占位而非估算。

#### E-03 报告详情弹窗使用 `dangerouslyUseHTMLString`，存在 XSS 风险（高）
- 证据：`frontend/src/views/Assignments.vue` 第 735 行。
- 用户影响：恶意文本可注入脚本，威胁账号与数据安全。
- 解决方案：
  - 流程：改为安全渲染组件，统一做 HTML 转义。
  - 交互：长文本采用可滚动纯文本展示。
  - 文案：过滤非法字符时提示“内容已安全处理”。
  - 容错：后端入库前加 XSS 清洗策略。

#### E-04 任务创建校验不足（中）
- 证据：`frontend/src/views/Assignments.vue` 第 615 行仅校验标题。
- 用户影响：缺失班级/日期合法性时，用户在提交后才收到后端报错。
- 解决方案：
  - 流程：前端补齐必填与日期先后校验（开始 <= 截止）。
  - 交互：按钮禁用直到校验通过。
  - 文案：字段级错误提示替代全局 toast。
  - 容错：后端返回字段错误时定位到对应控件。

## 3. 优化实施建议（落地排期）

### Phase 1（1-2 天，必须先做）
- 修复脚本阻断项：`init_db.py`、`reset_admin_password.py`。
- 修复安全与边界风险：E-01（提交角色限制）、E-03（XSS）。
- 建立统一错误码与前端提示映射（B-01、B-02）。

### Phase 2（2-4 天，核心体验）
